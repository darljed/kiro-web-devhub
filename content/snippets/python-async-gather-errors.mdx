## Overview

A structured concurrency utility for running batches of async tasks with bounded parallelism, individual timeouts, and comprehensive error collection. Unlike raw `asyncio.gather()`, it preserves order, collects partial results, and prevents one failure from losing all results.

## Code

```python
result = await gather_with_errors(
    tasks,
    max_concurrency=10,
    timeout=5.0,
)
print(f"Got {result.succeeded}/{result.total} results")
```

## Notes

- `max_concurrency` uses a semaphore to prevent overwhelming downstream services
- Individual task timeouts prevent one slow task from blocking the entire batch
- Results preserve original order via index tracking regardless of completion order
- The `with_retry` wrapper can be composed with `gather_with_errors` for resilient batch operations
- `return_exceptions=True` (default) prevents one failure from canceling other in-flight tasks
