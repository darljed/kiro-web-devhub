## Overview

A GitHub Actions CI/CD workflow designed for monorepo architectures that only builds, tests, and deploys packages that have actually changed. This eliminates wasted CI minutes and enables independent deployment of services, which is critical as monorepos grow beyond a handful of packages.

Key design decisions:
- **Path filtering** detects which packages changed, including shared dependency changes that affect downstream packages
- **Matrix strategy** runs tests in parallel per affected package for fast feedback
- **Concurrency control** cancels outdated runs when new commits are pushed
- **Conditional deployment** only deploys services whose code actually changed
- **Turborepo integration** leverages build caching and dependency-aware task execution
- **fail-fast: false** ensures one package failure does not skip tests for other packages

## Notes

- The `dorny/paths-filter` action compares against the base branch to detect changes accurately in PRs
- Include shared package paths in downstream filters (api, web) so shared changes trigger dependent builds
- Use `fail-fast: false` to get complete test results even when one package fails
- The `concurrency` group with `cancel-in-progress` saves CI minutes on rapid pushes
- For larger monorepos (10+ packages), consider Nx or Turborepo remote caching to further reduce build times
- Add `workflow_dispatch` trigger for manual deployments and rollbacks
- Use GitHub Environments with required reviewers for production deployment approval gates
