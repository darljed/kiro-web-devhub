## Overview

Comprehensive configuration management using ConfigMaps for non-sensitive data and Secrets for credentials, with multiple injection methods (env vars, env files, volume mounts) and automatic reloading on changes.

## Code

```yaml
env:
  - name: DB_URL
    valueFrom:
      secretKeyRef:
        name: app-secrets
        key: DATABASE_URL
volumeMounts:
  - name: config
    mountPath: /etc/app
```

## Notes

- `stringData` in Secrets accepts plain text (more readable than base64 `data`)
- Volume-mounted ConfigMaps update automatically when the ConfigMap changes (kubelet sync)
- Environment variables from ConfigMaps do NOT update on change (requires pod restart)
- Use Stakater Reloader or config hash annotations to trigger rollouts on config changes
- Never commit Secrets to git; use sealed-secrets, external-secrets-operator, or Vault CSI
- `optional: false` fails pod startup if the ConfigMap/Secret does not exist
