## Overview

A production retry decorator that handles both sync and async functions with exponential backoff, full jitter, exception filtering (retry only specific errors), and lifecycle callbacks for monitoring. Prevents thundering herd problems when services recover.

## Code

```python
@retry(max_attempts=3, retry_on=(TimeoutError,), base_delay=1.0)
async def unreliable_call():
    ...
```

## Notes

- Full jitter (`random.uniform(0, delay)`) is recommended over equal jitter by AWS architecture docs
- The `exclude` parameter prevents retrying non-transient errors (validation, auth, 4xx responses)
- `on_retry` enables metrics emission for monitoring retry frequency
- `on_give_up` triggers alerting when a dependency is truly down
- Automatically detects async vs sync and uses appropriate sleep mechanism
