## Overview

Enhanced dataclasses that validate data on construction via `__post_init__`, ensuring domain objects are always in a valid state. Value objects like `Email` and `Money` are frozen (immutable) to prevent accidental mutation.

## Code

```python
@dataclass(frozen=True)
class Money:
    amount: int  # cents
    currency: str = "USD"

    def __post_init__(self):
        if self.amount < 0:
            raise ValidationError("amount", "Cannot be negative")
```

## Notes

- `frozen=True` makes instances immutable (raises on attribute assignment)
- `__post_init__` runs after `__init__`, ideal for validation logic
- Store money as integers (cents) to avoid floating-point precision errors
- Value objects with validation ensure invalid data cannot propagate through the system
- Use `@classmethod` factories for alternative construction with different input formats
