    If chunk_size is set, yields lists of rows for batch processing.
    Supports .gz compressed files automatically.

                chunk.append(row)
                if len(chunk) >= chunk_size:
                    yield chunk
                    chunk = []
            if chunk:
                yield chunk
        else:
            yield from reader

## Overview

Memory-efficient patterns for processing CSV files of any size using generators. Data streams through transformations row-by-row (or chunk-by-chunk), enabling processing of multi-gigabyte files with constant memory usage and optional parallel execution.

## Code

```python
for row in stream_csv("data.csv.gz"):
    process(row)  # One dict at a time, constant memory

# Or in chunks for batch operations
for chunk in stream_csv("data.csv", chunk_size=1000):
    batch_insert(chunk)
```

## Notes

- Automatic gzip detection based on `.gz` extension works for both reading and writing
- The `transformer` returning `None` provides inline filtering without separate passes
- `DictReader` yields dictionaries with column-name keys for readable access
- Parallel processing uses `ProcessPoolExecutor` to bypass the GIL for CPU-bound transforms
- For extremely large files (100GB+), consider Apache Arrow or Polars for columnar processing
