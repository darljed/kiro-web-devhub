export const codeBlocks = {
  main: `| User session analysis with conversion funnel
index=application sourcetype=web_events
| transaction session_id maxspan=30m maxpause=5m
| eval session_duration=duration
| eval page_count=eventcount
| eval reached_cart=if(searchmatch("page=cart"), 1, 0)
| eval reached_checkout=if(searchmatch("page=checkout"), 1, 0)
| eval completed_purchase=if(searchmatch("event=purchase_complete"), 1, 0)
| stats
    count as total_sessions,
    avg(session_duration) as avg_session_sec,
    avg(page_count) as avg_pages,
    sum(reached_cart) as cart_sessions,
    sum(reached_checkout) as checkout_sessions,
    sum(completed_purchase) as purchase_sessions
| eval cart_rate=round((cart_sessions/total_sessions)*100, 1)
| eval checkout_rate=round((checkout_sessions/cart_sessions)*100, 1)
| eval purchase_rate=round((purchase_sessions/checkout_sessions)*100, 1)
| eval overall_conversion=round((purchase_sessions/total_sessions)*100, 2)

| Request flow through microservices
index=application trace_id=*
| transaction trace_id maxspan=10s
| eval total_duration_ms=duration*1000
| eval service_count=mvcount(service_name)
| eval services_visited=mvjoin(mvdedup(service_name), " -> ")
| eval had_error=if(searchmatch("status>=500 OR level=error"), 1, 0)
| stats
    count as total_traces,
    avg(total_duration_ms) as avg_trace_ms,
    perc95(total_duration_ms) as p95_trace_ms,
    sum(had_error) as error_traces,
    values(services_visited) as common_paths
    by services_visited
| sort - total_traces
| head 10

| Checkout process bottleneck detection
index=application sourcetype=checkout_events

| eval steps=mvcount(event_type)

## Overview

Transaction commands group related events into logical units for end-to-end analysis. This enables conversion funnel tracking, distributed trace analysis, and bottleneck detection across multi-step business processes.

## Code

```spl
| transaction session_id maxspan=30m maxpause=5m
| eval conversion=if(searchmatch("event=purchase"), 1, 0)
| stats avg(duration) as avg_session_time, sum(conversion)/count as rate
```

## Notes

- `maxspan` limits total transaction duration; `maxpause` limits gaps between events
- `searchmatch` within transactions searches across all events in the group
- `mvcount(event_type)` counts distinct steps in multi-event flows
- Transaction commands are memory-intensive; use `stats` with `earliest`/`latest` for better performance at scale
- For distributed tracing, correlate on `trace_id` rather than relying on time proximity
