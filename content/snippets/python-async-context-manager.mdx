## Overview

Async context managers for lifecycle management of shared resources. The connection pool pattern ensures connections are returned to the pool even when exceptions occur, while the distributed lock pattern provides mutual exclusion across async tasks or processes.

## Code

```python
async with managed_pool("postgres://...") as pool:
    async with pool.acquire() as conn:
        await conn.execute("SELECT 1")
```

## Notes

- `asynccontextmanager` from contextlib simplifies creation of async context managers
- The semaphore limits total concurrent connections regardless of pool/create balance
- Bad connections (those that raised) are discarded rather than returned to the pool
- `asyncio.timeout()` (Python 3.11+) provides clean timeout semantics with proper cancellation
- Stack multiple context managers with `async with` for composed resource acquisition
