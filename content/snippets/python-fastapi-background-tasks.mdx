class TaskQueue:
    """In-process async task queue with concurrency control."""

        self._max_workers = max_workers

## Overview

Background task patterns for FastAPI: lightweight fire-and-forget tasks via `BackgroundTasks`, a managed task queue with concurrency control for heavier work, and a periodic scheduler for recurring jobs. All integrate with FastAPI's lifespan for clean startup/shutdown.

## Code

```python
@app.post("/users")
async def create_user(request: Request, background_tasks: BackgroundTasks):
    background_tasks.add_task(send_email, user.email)

```

## Notes

- `BackgroundTasks` runs after the response is sent but within the same process (no persistence)
- The `TaskQueue` provides bounded concurrency for CPU/IO intensive operations
- The `Scheduler` replaces cron for simple periodic tasks within the application process
- FastAPI lifespan ensures clean startup and graceful shutdown of background workers
- For production workloads requiring persistence and retries, use Celery, ARQ, or Dramatiq
