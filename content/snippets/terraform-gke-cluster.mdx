## Overview

A production GKE cluster with private nodes, Workload Identity for secure GCP API access, network policies, cluster autoscaling, and separate node pools for general workloads and cost-optimized spot instances.

## Code

```hcl
module "gke" {
  source       = "./modules/gke"
  project_id   = "my-project"
  cluster_name = "production"
  region       = "us-central1"
}
```

## Notes

- Private cluster nodes have no public IPs, reducing attack surface
- Workload Identity maps Kubernetes service accounts to GCP service accounts (no key files)
- Spot node pool with taints ensures only tolerant workloads (batch jobs) schedule there
- Maintenance windows restrict auto-upgrades to weekends during low traffic
- `REGULAR` release channel provides stable versions with security patches
