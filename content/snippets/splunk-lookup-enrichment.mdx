index=application sourcetype=access_combined
| lookup customer_segments.csv customer_id OUTPUT segment, tier, account_manager

## Overview

Lookup enrichment patterns that augment raw log data with business context from CSV files and KV stores. Includes building dynamic baselines via `outputlookup` and using them for deviation detection.

## Code

```spl
| lookup customer_segments.csv customer_id OUTPUT segment, tier
| lookup oncall_schedule.csv team OUTPUT primary_oncall
| eval priority=case(tier="platinum", "P1", tier="gold", "P2")
```

## Notes

- `coalesce()` provides default values when lookup matches are missing
- `outputlookup` creates/updates CSV files from search results for use in future searches
- KV store lookups support real-time updates without reindexing
- Multi-field lookups match on composite keys for precise enrichment
- Schedule baseline-building searches to run daily and store results for comparison queries
