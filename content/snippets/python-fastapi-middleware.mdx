class RequestContextMiddleware(BaseHTTPMiddleware):
    """Inject request ID and timing into every request."""

    async def dispatch(
        self, request: Request, call_next: RequestResponseEndpoint
    ) -> Response:
        # Generate or extract request ID
        request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))
        request_id_var.set(request_id)

        # Start timing
        start_time = time.perf_counter()

        try:

        except Exception as exc:

## Overview

Production middleware stack for FastAPI that handles cross-cutting concerns: request ID propagation for distributed tracing, response timing, structured logging with contextual data, and IP-based rate limiting. Uses `contextvars` for request-scoped data that flows through the entire call stack.

## Code

```python
app.add_middleware(RateLimitMiddleware, requests_per_minute=100)
app.add_middleware(RequestContextMiddleware)
```

## Notes

- Middleware executes in reverse registration order (last added runs first)
- `contextvars` provide thread-safe, async-safe request-scoped state without passing request objects
- The `X-Request-ID` header enables distributed tracing across microservices
- For production rate limiting, replace the dict with Redis for multi-instance support
- `structlog` produces JSON logs compatible with DataDog, Splunk, and CloudWatch
