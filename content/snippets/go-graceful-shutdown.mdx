## Overview

A production-ready Go HTTP server that handles shutdown gracefully, ensuring all in-flight requests complete before the process exits. This is critical for zero-downtime deployments in containerized environments where orchestrators (Kubernetes, ECS) send SIGTERM before stopping a pod.

Key design decisions:
- **Signal handling** captures SIGINT and SIGTERM for clean shutdown
- **Health endpoint** returns 503 during shutdown so load balancers stop sending traffic
- **Request tracking** uses atomic counters to monitor active connections
- **Configurable timeout** prevents indefinite hangs from stuck connections
- **Keep-alive disabling** ensures no new requests arrive on existing connections

## Notes

- Adjust the 5-second load balancer detection delay based on your infrastructure (AWS ALB needs ~10s, Kubernetes needs 1 pod deregistration cycle)
- The `IdleTimeout` of 120s matches common load balancer defaults; align with your LB configuration
- For HTTP/2, the server automatically sends GOAWAY frames during shutdown
- Consider adding a `/readyz` endpoint separate from `/healthz` for startup vs runtime health
- In Kubernetes, set `terminationGracePeriodSeconds` to be longer than your `shutdownTimeout`
- Use `httpServer.RegisterOnShutdown()` to close WebSocket connections or database pools
