## Overview

A builder pattern that uses TypeScript's type system to enforce required fields at compile time. The `build()` method is only available when all required fields have been set, preventing runtime errors from incomplete configurations.

## Code

```typescript
class HttpRequestBuilder<State extends BuilderState = {}> {
  url(url: string): HttpRequestBuilder<State & { url: true }> {
    this.config.url = url;
    return this as unknown as HttpRequestBuilder<State & { url: true }>;
  }

  method(method: HttpRequestConfig["method"]): HttpRequestBuilder<State & { method: true }> {
    this.config.method = method;
    return this as unknown as HttpRequestBuilder<State & { method: true }>;
  }

  // build() is only callable when State includes both url and method
  build(this: HttpRequestBuilder<{ url: true; method: true }>): HttpRequestConfig {
    return this.config as HttpRequestConfig;
  }
}
```

## Notes

- The `State` generic parameter tracks which required fields have been set
- `build()` uses a `this` parameter type to restrict when it can be called
- Optional methods like `header()`, `timeout()` return `this` without modifying the state type
- This pattern eliminates "required field missing" runtime errors entirely
- Works well for complex configuration objects with many optional and few required fields
