## Overview

A production retry decorator that implements exponential backoff with optional jitter. This is essential for any code that calls external services, as transient failures (network timeouts, rate limits, temporary unavailability) are inevitable in distributed systems.

Key design decisions:
- **Jitter** prevents multiple clients from retrying at the same time (thundering herd problem)
- **Exception filtering** ensures you only retry on transient errors, not on permanent failures like validation errors
- **Max delay cap** prevents unreasonably long waits during extended outages
- **Callback hook** enables custom telemetry or alerting on retries

## Code

```python
import time
import random
import functools
import logging
from typing import Callable, Type

logger = logging.getLogger(__name__)


def retry(
    max_attempts: int = 3,
    base_delay: float = 1.0,
    max_delay: float = 60.0,
    exponential_base: float = 2.0,
    jitter: bool = True,
    exceptions: tuple[Type[Exception], ...] = (Exception,),
    on_retry: Callable[[Exception, int], None] | None = None,
):
    """Retry decorator with exponential backoff and optional jitter."""

    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exception: Exception | None = None

            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    last_exception = e
                    if attempt == max_attempts:
                        logger.error(f"{func.__name__} failed after {max_attempts} attempts: {e}")
                        raise

                    delay = min(base_delay * (exponential_base ** (attempt - 1)), max_delay)
                    if jitter:
                        delay = delay * (0.5 + random.random())

                    logger.warning(f"{func.__name__} attempt {attempt}/{max_attempts} failed. Retrying in {delay:.2f}s...")
                    if on_retry:
                        on_retry(e, attempt)
                    time.sleep(delay)

            raise last_exception or RuntimeError("Retry logic error")
        return wrapper
    return decorator
```

## Notes

- For async functions, create an `async_retry` variant that uses `asyncio.sleep` instead of `time.sleep`
- Common exception tuples: `(ConnectionError, TimeoutError)` for network calls, `(IOError,)` for file operations
- Set `jitter=False` only in tests where you need predictable timing
- The `on_retry` callback is useful for incrementing metrics counters or sending alerts
- Consider adding `retry_if` parameter that accepts a predicate function for more complex retry conditions
- For HTTP clients, also check the response status code (e.g., retry on 429, 502, 503)
