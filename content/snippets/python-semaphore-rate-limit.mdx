## Overview

Async rate limiting using token bucket and semaphore patterns for controlling throughput to external APIs. Combines rate limiting (tokens per second) with concurrency limiting (max parallel requests) to prevent overwhelming downstream services.

## Code

```python
limiter = AsyncRateLimiter(rate=10, max_tokens=20)  # 10/sec, burst 20
await limiter.acquire()  # Waits if needed
```

## Notes

- Token bucket allows initial bursts up to `max_tokens` while enforcing sustained `rate`
- `ConcurrencyLimiter` caps parallel requests independently of rate (protects connection pools)
- The `rate_limited` decorator combines both for comprehensive protection
- Use `time.monotonic()` instead of `time.time()` to be immune to system clock adjustments
- For distributed rate limiting across multiple processes, use Redis-based implementations
