## Overview

Redis Pub/Sub for fire-and-forget broadcasting and Redis Streams for reliable, persistent event delivery with consumer groups. Streams provide at-least-once delivery, message acknowledgment, and dead letter handling.

## Code

```
-- Fire and forget
PUBLISH channel '{"event":"data"}'

-- Reliable delivery with consumer groups
XADD stream * key value
XREADGROUP GROUP mygroup consumer1 STREAMS stream >
XACK stream mygroup message-id
```

## Notes

- Pub/Sub is fire-and-forget: if no subscriber is connected, the message is lost
- Redis Streams persist messages and support consumer groups for reliable processing
- `BLOCK 5000` makes XREADGROUP wait up to 5 seconds for new messages (long polling)
- `XAUTOCLAIM` recovers messages from crashed consumers after a timeout (dead letter handling)
- `MAXLEN ~` with approximate trimming is more efficient than exact trimming
- For cross-service communication requiring guaranteed delivery, consider RabbitMQ or Kafka instead
