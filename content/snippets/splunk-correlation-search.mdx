| eval journey_steps=mvcount(sourcetype)

| eval has_payment=if(searchmatch("sourcetype=payment"), 1, 0)
| stats

    sum(has_error) as error_sessions,
    sum(has_payment) as payment_sessions,
    avg(duration) as avg_session_duration
| eval conversion_rate=round((payment_sessions/total_sessions)*100, 2)
| eval error_impact_pct=round((error_sessions/total_sessions)*100, 2)

## Overview

Correlation searches that join events across multiple indexes and source types to establish causality between infrastructure changes, deployments, and application errors. These are essential for reducing mean time to resolution (MTTR) during incidents.

## Code

```spl
index=application status>=500
| join type=left host [search index=infrastructure cpu_pct>90]
| where isnotnull(cpu_pct)
```

## Notes

- Use `transaction` for grouping events by session/request ID across sources
- `join` with time window correlation identifies causal relationships (deploy then errors)
- Keep correlation windows tight (5-15 min) to reduce false positives
- `dc()` (distinct count) helps identify distributed vs. single-source attack patterns
- Schedule these as saved searches for automatic incident enrichment
