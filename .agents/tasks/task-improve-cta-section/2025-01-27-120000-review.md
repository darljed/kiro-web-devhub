# Vibrant gradient and animated SVG overhaul for CTA section

The CTA section moves from a muted zinc-900/800 card to a purple-indigo-sky gradient with six inline SVG decorative elements animated via CSS keyframes. The approach uses inline `style` attributes for animation binding rather than Tailwind utility classes, and adds four `@keyframes` definitions to the global stylesheet. Semantic HTML, the AnimateOnScroll wrapper, heading text, and the `/snippets` link are preserved.

**Watch for:** Duplicate keyframes definitions (confirmed), missing `prefers-reduced-motion` handling for six continuously-animating elements (confirmed), and dropped dark-mode variant on the card background (confirmed).

## High-level view

The old dark-on-dark card becomes a saturated gradient with depth cues (shadow-2xl, larger glow blurs). Six SVGs at absolute positions provide visual motion through float, spin, rotate, and pulse animations.

The animation system lives in `globals.css` as raw `@keyframes` rules. Two of the four keyframe blocks (`rotate` and `spin-slow`) are identical, producing the same 0-360deg rotation. The differentiation happens entirely in the `style` prop (duration values), making one of the keyframes redundant.

The old component had a `dark:from-zinc-800 dark:to-zinc-900` variant. The new gradient (`from-purple-700 via-indigo-700 to-sky-600`) has no dark variant, so it renders identically in both modes — a change in behavior from the original.

All six SVGs animate continuously with no pause, and the codebase has no `prefers-reduced-motion` media query anywhere. Users with motion sensitivity enabled will still see spinning, floating, and pulsing elements.

<summary>Issues (4)</summary>

1. **Duplicate keyframes (`rotate` vs `spin-slow`)** — Both produce identical `0deg -> 360deg` rotation. Remove one and use the remaining keyframe name with different durations in the `style` prop where needed.
2. **No `prefers-reduced-motion` respect** — Six continuously-animating SVGs ignore the user's motion preference. Add a `@media (prefers-reduced-motion: reduce)` block that sets `animation: none` on these elements.
3. **Dark-mode gradient variant dropped** — The old card had `dark:from-zinc-800 dark:to-zinc-900`; the new gradient has no dark variant. Verify this is intentional, or add a subtle dark-mode adjustment (e.g., slightly deeper purple).
4. **Inline `style` for animations instead of Tailwind** — Using `style={{ animation: "..." }}` bypasses Tailwind's utility model. Consider defining custom animation utilities in `@theme inline` or at minimum documenting why inline styles are necessary here.

## Duplicate keyframes definitions

The `rotate` and `spin-slow` keyframes in `globals.css` are byte-for-byte identical:

```css
@keyframes rotate {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}

@keyframes spin-slow {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}
```

The only difference in behavior comes from the inline style durations: `rotate 6s linear infinite` vs `spin-slow 8s linear infinite`. One keyframe named `rotate` (or even Tailwind's built-in `spin`) would suffice, with different `animation-duration` values in the `style` prop.

## Accessibility: motion sensitivity

The six SVGs use `float 3s`, `spin-slow 8s`, `pulse-scale 2.5s`, `float 4s`, `rotate 6s`, and `pulse-scale 3s` — all infinite, all running simultaneously. A single rule at the bottom of `globals.css` would address this:

```css
@media (prefers-reduced-motion: reduce) {
  svg[aria-hidden="true"] {
    animation: none !important;
  }
}
```

Since the SVGs are purely decorative (`aria-hidden="true"`), disabling their animation loses zero functional information.

## Dark-mode behavior change

The old card explicitly adjusted for dark mode with `dark:from-zinc-800 dark:to-zinc-900`, producing a lighter card against the `#0a0a0a` page background. The new gradient is a fixed purple-to-sky sweep regardless of mode — a departure from the pattern where the card adapts. The subtitle text color also changed from `text-zinc-300` (neutral) to `text-indigo-100/90` (tinted), which would appear slightly odd if the gradient were ever removed or made conditional.

## Animation binding via inline styles

All six SVGs use `style={{ animation: "float 3s ease-in-out infinite" }}` rather than Tailwind classes. The project uses Tailwind CSS 4 with `@theme inline`, which supports custom animation utilities. The inline style approach means animation behavior isn't visible in the class list during visual debugging, and bypasses Tailwind's design system for consistency if more animated elements are added later.
